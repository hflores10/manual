Comandos básicos de Git

Empezar a monitorear proyecto, carpeta o repositorio
git init

Ver el estado del proyecto, si hay cambios por guardar,
archivos o carpetas nuevos.
git status

Agregar archivos al repositorio
git add nombre_de_archivo / git add file_name
(Agrega un archivo). Para agregar todos los archivos usa
git add -A

Guardar los cambios que hicimos en el repositorio
git commit -m "este_es_un_mensaje"

Listamos los commits con su información respectiva
git log
Nos dará autor, fecha, código hash, los mensajes aparecen
en el final de la descripción.

Para viajar en el tiempo entre commits o ramas
git checkout codigo_hash
Movernos al último commit que tenemos
git checkout master
git reset es parecido a git checkout con la diferencia de que
borra los commits:

git reset
git reset --soft
No toca el Working Area / No borra el código
git reset --mixed
No toca el working Area pero borra el Staging Area
git reset --hard
Borra absolutamente todo lo que hay en el commit

Para clonar un repositorio remoto:
git clone https://www.github.com/hflores07/manual.git

Para ignorar archivos o carpetas:
creas en la carpeta del proyecto un archivo ".gitignore" y
dentro de este archivo nombramos los archivos o carpetas a ignorar

Para descartar cambios en un directorio de trabajo:
git checkout -- nameOfFile

Para que sirve:
git branch -M main

Para crear y extraer una nueva rama:
git checkout -b nueva_rama

Para crear y extraer una rama distinta del HEAD actual:
git checkout -b nueva_rama rama_existente

Para configurar nombre de usuario y correo
git config --global user.name usuario_nombre
git config --global user.email usuario_@correo

Para fusionar los cambios con la rama master, main, etc
Las fusiones pueden ser fast-fordware (simples y automáticas. Pasa cuando trabajas con archivos y lineas de codigo distintas).
Manual merge (largo y manual. Pasa cuando se trabaja sobre el mismo archivo y lineas de código).
desde la rama que absorverá los cambios:
git merge rama_de_procedencia_de_cambios

Para subir los cambios a un repositorio remoto:
git remote add origin URL_del_repositorio
git remote (Ver repositorio)
git remote -v (Ver links de repositorios remotos)
git push -u origin master / git push origin master
